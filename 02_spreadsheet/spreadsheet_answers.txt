Cleaning Steps
To ensure data consistency across the dataset, the following cleaning steps were performed:

Status Column: Applied =TRIM(PROPER(F2)) to remove leading/trailing spaces and standardize inconsistent casing (e.g., converting "captured" and "CAPTURED" to "Captured").

Gateway Region: Used =UPPER(TRIM(I2)) to ensure all regional codes (APAC, EU, US) were uniform and free of accidental whitespace.

Risk Score: Stripped non-numeric text (like "score:") using string functions to ensure the column contained only valid integers for analysis.

Standardization Rules
Handling missing or "noisy" data was critical for accurate reporting:

Empty Risk Scores: Missing values in the risk_score column were assigned a "Worst-Case" score of 80. In a fintech environment, unknown risk is treated as high risk to ensure these transactions are flagged for manual review.

Unknown Regions: Any missing values in the gateway_region were labeled as "UNKNOWN" to avoid exclusion from regional pivot tables.

Lookup and Enrichment Logic
The dataset was enriched using two primary external sources:

Currency Conversion: Used a VLOOKUP against the Exchange_Rates tab to find the appropriate multiplier for each currency (INR, EUR, etc.) to calculate a standardized amount_usd.

Merchant Details: Merged data from merchant_master.csv using VLOOKUP to associate each transaction with its corresponding Merchant Category and Rating.
